Understanding Callbacks

There are 4 callbacks at various points in the auth cycle available.

Available Callbacks


  • after_set_user

  • after_authentication

  • after_authorization

  • before_logout

Callbacks are very useful for modifying or collecting stats/info on your users.

  1. after_set_user

    This is called every time the user is set. The user is set:

    • when the user is initially authenticated.
    • when the user is set via Warden::set_user().

    Example:


    Warden::after_set_user(function($user) {
  logger(\Fuel::L_INFO, 'User '.$user->id.' was set as current user', 'Warden::after_set_user');
});

  2. after_authentication

    This is called every time the user is authenticated.

    Example:


    Warden::after_authentication(function($user) {
  // Log to file or log some stats to Redis etc..
  logger(
    \Fuel::L_INFO,
    'User '.$user->id.' was authenticated successfully at '.$user->current_sign_in_at,
    'Warden::after_authentication'
  );
});

  3. after_authorization

    This is called every time the user is successfully authorized. The user is authorized when:

    • Warden::can() is successful.
    • Warden::authorize() is successful (ie. does not throw an Exception).

    Example:


    Warden::after_authorization(function($user) {
  logger(
    \Fuel::L_INFO,
    'User '.$user->id.' was successfully authorized to access '.\Input::server('REQUEST_URI')
  );
});

  4. before_logout

    This is called before each user is logged out.

    • when the user is initially authenticated.
    • when the user is set via Warden::set_user().

    Example:


    Warden::before_logout(function($user) {
  logger(\Fuel::L_INFO, 'User '.$user->id.' logging out', 'Warden::before_logout');
});
