More on Warden Tasks

Warden comes bundled with tasks to help you install and/or add and remove features.

Adding a Warden feature

Issuing one of the commands on the right will add the required fields to your users table or create a new table if the feature doesn't use the users table. It will also activate the feature in your config.

You can also add multiple features in the same command, eg.:
php oil r warden --omniauthable --confirmable

This will do the following:

  • Enable the omniauthable and confirmable features in your APPPATH/config/warden.php. Creating the file if it doesn't exist.
  • Creating the services table if it doesn't exist for omniauthable.
  • Add the confirmable fields to the users table if they don't exist.

To add a Warden feature to an existing installation, use the command line to issue the following commands:

  • recoverable:

    php oil r warden --recoverable or php oil r warden -r

  • confirmable:

    php oil r warden --confirmable or php oil r warden -c

  • trackable:

    php oil r warden --trackable or php oil r warden -t

  • lockable:

    php oil r warden --lockable or php oil r warden -l

  • omniauthable:

    php oil r warden --trackable or php oil r warden -t

  • profilable:

    php oil r warden --profilable or php oil r warden -p

Removing/Deleting a Warden feature

To remove a feature use one of the commands above followed by the option: -d or --delete

For example to remove the lockable feature, just issue the following command:

php oil r warden --lockable --delete or php oil r warden --lockable -d

This will remove the lockable fields from your users table and deactivate the feature in your config.
